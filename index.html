Cloud Computing & DevOps Fundamentals

AWS-Focused • 5 Days • 30 Hours

📅 5 Days
⏰ 3 Sessions per Day
🕒 2 Hours per Session
🎯 Total: 30 Hours
Day 1: Cloud Foundations & AWS Introduction
Session 1 – Cloud Fundamentals (2 hours)
  • Evolution of computing (Traditional vs Cloud)
  • Cloud service models (IaaS, PaaS, SaaS)
  • Deployment models (Public, Private, Hybrid)
  • Benefits of cloud (scalability, elasticity, cost-efficiency)
Session 2 – Cloud Ecosystem & Providers (2 hours)
  • Overview of major cloud providers
  • Introduction to AWS
  • Use cases of cloud in real-world scenarios
Session 3 – AWS Basics & IAM (2 hours)
  • AWS Global Infrastructure (Regions, Availability Zones)
  • AWS Management Console walkthrough
  • Identity and Access Management (IAM): users, roles, policies
Day 2: Core AWS Services & Database Fundamentals
Session 1 – Compute Services (EC2) (2 hours)
  • Introduction to Amazon EC2
  • Instance types, key pairs, security groups
Session 2 – Storage Services (S3) (2 hours)
  • Amazon S3 overview
  • Storage types and use cases
  • Hands-on: basic S3 usage
Session 3 – Database Fundamentals (2 hours)
  • What is a database?
  • SQL vs NoSQL (differences and use cases)
  • Overview of Amazon RDS and Amazon DynamoDB
  • Choosing the right database
Day 3: Networking (VPC) & DevOps Introduction
Session 1 – Networking Basics (2 hours)
  • Fundamentals of networking (IP, subnets)
  • Introduction to Amazon VPC
Session 2 – VPC Components & Security (2 hours)
  • Public vs Private subnets
  • Internet Gateway (concept)
  • Security Groups vs Network ACLs
  • Activity: AWS architecture diagram (VPC + EC2 + subnet)
Session 3 – DevOps Fundamentals (2 hours)
  • What is DevOps?
  • Traditional vs DevOps workflow
  • DevOps lifecycle (Plan → Code → Build → Test → Deploy → Monitor)
  • Key principles: automation, collaboration, continuous improvement
Day 4: Version Control & CI/CD Pipelines
Session 1 – Version Control (Git & GitHub) (2 hours)
  • Introduction to Git and GitHub
  • Basic commands (clone, commit, push)
  • Branching and collaboration workflows
Session 2 – Modern DevOps Concepts (2 hours)
  • Containerization (basic concept)
  • Orchestration (basic concept)
Session 3 – CI/CD Fundamentals & Hands-on (2 hours)
  • Continuous Integration vs Continuous Deployment
  • Pipeline stages (build, test, deploy)
  • Tools overview (GitHub Actions / Jenkins)
  • Hands-on: Create a simple CI/CD pipeline
Day 5: Monitoring, Cost Optimization & Mini Project
Session 1 – Monitoring & Logging (2 hours)
  • Importance of monitoring in cloud
  • Introduction to Amazon CloudWatch
  • Metrics vs Logs
Session 2 – Advanced AWS Services & Cost Optimization (2 hours)
  • AWS Lambda (serverless concept)
  • Amazon SNS (notifications)
  • Amazon CloudFront (CDN basics)
  • AWS pricing model (pay-as-you-go)
  • Billing dashboard and budget alerts
  • Cost optimization best practices
Session 3 – Mini Project & Presentation (2 hours)
  • Deploy a simple application on AWS
  • Use EC2/S3 with basic CI/CD
  • Group work, presentation, feedback
